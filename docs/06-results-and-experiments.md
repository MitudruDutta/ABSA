## 6.1 Headline results (RoBERTa, locked test, 27,000 cells)

| metric | score | note |
|---|---|---|
| **macro-F1 (3-class, pooled)** | **0.843** | the honest, imbalance-robust headline |
| aspect-detection F1 (present vs not) | 0.864 | the routing signal |
| polarity accuracy (neg vs neu, where present) | 0.986 | sentiment correctness |
| weighted-F1 / accuracy | 0.944 | **inflated by the 80% `not_present` majority** |

Per-class F1: not_present 0.967, negative 0.885, **neutral 0.677** (the weak point).

### Which number is the "true" one?
For an **imbalanced** problem, **macro-F1 (0.843) is the right headline** — it weights
all three classes equally and is not fooled by the majority class. Accuracy/weighted-F1
(0.944) are flattered by the fact that 80% of cells are the easy `not_present` class (a
trivial "always not_present" model already scores ~0.80 accuracy). The README leads with
macro-F1 for exactly this reason.

## 6.2 Two valid macro-F1 numbers (0.740 vs 0.843)

- **0.740** = average the 9 per-aspect macro-F1s separately (dragged down by `bug` =
 0.587, whose `neutral` class is near-empty).
- **0.843** = pool all 27k cells, then compute macro-F1 once.

Both are legitimate and answer different questions; **pooled (0.843) is the standard
ABSA report.** Neither is fabricated.

## 6.3 Verification — the score was independently proven

Because 0.843 looked high, it was **re-derived by hand** inside the notebook:

1. build the confusion matrix (raw counts),
2. compute precision/recall/F1 per class with plain numpy from the matrix,
3. average → manual macro-F1,
4. compare to `sklearn.classification_report`.

```
MANUAL macro-F1 : 0.843
sklearn macro-F1 : 0.843
match : True
```

The confusion matrix also makes the imbalance and the errors visible:

```
 pred:not_present pred:negative pred:neutral
gold not_present 20953 560 187
gold negative 409 3969 21 ← 409 missed complaints (9% of negatives)
gold neutral 288 45 568 ← neutral recall only 0.63
```

The most business-relevant error is **negative → not_present (409)**: real complaints
the model would fail to route — 9% of all complaints.

## 6.4 Score progression

| stage | test mean per-aspect macro-F1 |
|---|---|
| TF-IDF baseline | 0.724 |
| RoBERTa + weighted loss | 0.736 |
| per-aspect ensemble | 0.740 |

Scaling labels 2.5k → 20k lifted **both** models and **shrank** the TF-IDF↔RoBERTa gap.
The finding: on clean synthetic text at scale, a linear baseline nearly matches a
fine-tuned transformer; the transformer's edge would widen on noisier real tickets.

## 6.5 Ensemble

For each aspect, pick the model that wins on **val** (never peeking at test) — TF-IDF
for `bug`/`hardware`, RoBERTa for the other 7 — then score the combination on test.
**0.740** mean per-aspect macro-F1, a small free bump that captures each model's strength.

## 6.6 Novel experiment — shared model vs per-queue specialists

**The project's headline finding.** Does one shared model match N per-queue specialists?
A separate model was trained per queue (TF-IDF for both, so it is feasible on small
queues a transformer can't fit) and compared to the single shared model.

| | mean aspect macro-F1 | models to maintain |
|---|---|---|
| **Shared (1 model)** | **0.713** | 1 |
| Per-queue specialists | 0.676 | 10 |

**The shared model wins on 10/10 queues (+0.037) at 1/10th the maintenance.** The gap
widens on small queues (Sales −0.121, HR −0.074): specialists **starve** when they
fragment scarce data, while the shared model transfers cross-aspect signal across all
queues. **Specialization hurts here** — a counterintuitive, measured, defensible result.
